# X/Twitter Spam Blocker 如何工作

X/Twitter Spam Blocker 会在支持的 X/Twitter 响应渲染到浏览器之前进行过滤。它从响应中解析推文和用户资料数据，匹配垃圾信号，然后判断命中的内容应该被隐藏，还是进入拉黑审核。

## 内联响应过滤

当 X/Twitter 加载支持的时间线、通知、帖子详情、搜索结果、书签或用户推文视图时，扩展会在页面渲染前处理对应响应。

扩展会从响应中读取垃圾过滤所需的推文内容、作者资料字段、链接和相关元数据。随后页面会收到过滤后的响应，因此被移除的项目不会出现在你的本地浏览视图中。

## 基于规则的多维度匹配

扩展会基于规则匹配推文和用户的多维度信号。判断时可能会参考回复文本、外部链接、个人简介、主页链接、账号元数据和重复出现的高风险模式。

这让扩展可以区分较低风险的内容清理和较高风险的账号处理。某条推文可能只会从本地视图中移除；而更强的匹配，尤其是同时涉及推文行为和账号资料信号时，则可能生成拉黑审核候选。

## 隐藏与拉黑审核

扩展主要有两类过滤结果：

`隐藏` 会在 X/Twitter 渲染前把命中的内容从响应中移除。该内容会从你的本地浏览视图中消失，并且可以记录在活动记录里。

`拉黑审核` 会移除命中的内容，同时为对应账号生成审核候选。该账号会出现在浮动审核面板中，由你决定是否拉黑。

候选账号不会被自动拉黑。只有在你从审核面板中选择账号并确认后，扩展才会提交拉黑操作。

## 本地记录

扩展会保存保护概览、活动记录、设置和安全限制所需的本地记录。这些记录可能包括：

- 保护统计。
- 隐藏内容记录。
- 高风险账号候选。
- 成功拉黑操作。
- 拉黑失败尝试。
- 每日拉黑安全状态。

当前版本中，设置、统计、活动历史和安全状态会保存在你的设备上。

## 网络与权限边界

扩展只会在支持的 X/Twitter 页面上处理相关内容。它不会处理无关网站。

对 `abs.twimg.com` 的访问仅用于读取准备 X 请求校验值所需的 X 网页资源，以便完成用户确认的拉黑请求。

当你确认拉黑账号时，拉黑请求会通过 X/Twitter 完成。扩展不会把这个动作转交给无关第三方服务。

## 不执行远程代码

扩展使用的 JavaScript 会随扩展一起打包。当前版本不会从远程服务器加载 JavaScript 或 Wasm，不会注入外部脚本文件，也不会执行远程获取的代码。

## 它不会做什么

X/Twitter Spam Blocker 不会删除 X/Twitter 上的原始内容，不会替其他用户隐藏内容，也不会保证识别所有垃圾内容。

它的目标是在你的浏览器里减少命中的垃圾干扰，并在拉黑前提供人工审核。
